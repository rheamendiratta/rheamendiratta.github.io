Style Guide — v2

Rhea Mendiratta's Website

A design system built on warm neutrals, an indigo primary accent, and three warm "spice" colors that mark content categories — research, code, and ideas. Minimal and editorial at its core, with hand-drawn doodles and hidden easter eggs.

01

Design Principles

✦

Clarity First

Every element earns its place. Active voice, low jargon, generous whitespace. If it doesn't help the reader, remove it.

✺

Warm Precision

Analytical rigor meets human warmth. The indigo accent, serif headings, and soft backgrounds signal approachability without sacrificing sharpness.

✿

Delightful Details

Hand-drawn doodles in margins, hover micro-interactions, and hidden easter eggs reward exploration. The site should make visitors smile.

Doodles & Easter Eggs

The playful layer that makes this site unmistakably yours. Doodles always use the indigo primary — they're the unifying thread across pages. They should feel hand-drawn and imperfect.

Easter Egg Ideas

☆ Konami code: Entering ↑↑↓↓←→←→BA reveals a hidden "fun facts" panel about you.

☆ Hover doodles: Section headings sprout small hand-drawn flourishes on hover — a tiny star, a squiggle underline, a flower.

☆ Cursor trail: A subtle sparkle or small dot trail follows the cursor on the home page — gentle, not distracting.

☆ 404 page: A full-character illustration with a message like "This page wandered off to get chai. Try going home?"

☆ Click the accent dot: The indigo dot in the nav, when clicked, cycles through the four accent colors (neel → kesar → gulabi → haldi) as a little color demo.

☆ Secret /uses page: A list of tools, apps, and gear you use daily — discoverable only through the footer or a hidden link.

07

Implementation Notes

Recommended Stack
Next.js or Astro — both support static generation, dark mode, and MDX for blog posts. Astro is lighter if you want pure content; Next.js gives you more room to grow if you add interactive features.
Framework:   Next.js 14+ or Astro 4+
Styling:     Tailwind CSS + CSS variables
Content:     MDX for blog posts
Fonts:       Google Fonts (self-hosted for perf)
Deploy:      Vercel (free tier is plenty)
Doodles:     SVG inline, animated with CSS
Dark mode:   CSS custom properties + JS toggle
Color system implementation: Define all four accent colors as CSS variables. The spice colors only need two variants each (full + soft/transparent). Use a data attribute or class on project/writing cards to set the category, and let CSS handle the tag coloring. This keeps the markup clean and the colors easy to update.
Performance: Self-host fonts for speed. Keep doodle SVGs inline so they respond to the --doodle-color variable. Lazy-load project card images. Target under 100KB total CSS+JS for instant loads.
